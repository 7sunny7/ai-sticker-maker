# AI Sticker Maker MVP 产品方案

## 目标

第一版只做一个闭环：

> 用户用文字生成一套原创角色微信表情包，按积分出图，完成 8 张后导出基础投稿包。

优先级：

1. 质量稳定
2. 积分消耗可控
3. 用户操作尽量少

## 目标用户

第一版先面向创作者，不主打普通个人照片转表情。

主要用户：

- 微信表情投稿创作者
- 小红书、视频号、公众号等轻 IP 创作者
- 想给账号做统一角色表情包的运营者

暂不主打：

- 宠物照片转表情
- 宝宝照片转表情
- 情侣/真人照片转表情

上传角色参考图放到第二版。

## 产品形态

第一版先做网页 Web App，不先做微信小程序。

原因：

- 图片预览、文件下载、ZIP 投稿包导出更适合网页。
- 小程序在 AI 生成内容、积分支付、文件下载和外链跳转上限制更多。
- 微信表情开放平台本身是网页投稿流程。
- MVP 阶段需要频繁调整流程和积分规则，网页迭代更快。

## MVP 核心流程

### 1. 输入主题

用户输入一句描述，例如：

```text
一只嘴贱的加班橘猫，适合微信聊天
```

如果用户不清楚想做什么，可以先抽方向卡。

方向卡只生成低成本结构，不出图。每张方向卡包含：

- 主角
- 性格
- 使用场景
- 画风倾向
- 表情套组方向

用户选择方向卡后，再进入完整方案生成。

方向卡支持刷新。刷新只更换创作方向，不生成图片，不消耗积分，也不影响用户已经选择的表情数量。

### 2. 生成表情方案

系统免费生成结构化方案，不直接出图。

方案包含：

- 主角设定
- 画风设定
- 8 个表情含义词
- 每张表情动作描述
- 整套生成规范
- 禁止项

MVP 不生成表情包名称。

### 3. 编辑和确认方案

用户可以编辑：

- 主角描述
- 画风
- 8 个含义词
- 单张动作描述

这一步不消耗图像积分。

### 4. 生成角色候选

默认生成 3 张角色候选图。

规则：

- 每张角色候选图正常扣积分。
- 用户选择 1 张作为角色基准。
- 不把角色候选做成免费权益，避免用户白嫖图片。

### 5. 生成角色卡

系统根据用户选中的角色生成结构化角色卡。

角色卡包含：

- 外形特征
- 颜色
- 服装/配件
- 表情风格
- 不可变化点

后续所有表情图都基于角色卡生成。

### 6. 分张生成表情

MVP 不采用“一张大图生成所有表情再切图”。

原因：

- 单张失败会拖累整套。
- 网格切图不稳定。
- 单个 prompt 描述 8 个动作容易混淆。
- 用户不满意某一张时，整套重做会导致积分消耗不可控。

第一版采用：

- 一张表情一张图。
- 先生成 2 张代表性表情，确认角色和风格稳定。
- 用户确认后，再生成剩余 6 张。
- 每张图都正常扣积分。

这里的“先生成 2 张”只是分批生成，不是低价小样。

### 7. 单张重生成

只支持单张重生成，不默认支持整套重做。

重生成前让用户选择原因：

- 角色不像
- 表情不对
- 画面不好看
- 太复杂
- 其他

系统根据原因优化提示词后再生成，避免盲目抽卡。

### 8. 基础质量检查

每张图生成后做基础检查：

- 是否像同一角色
- 主体是否完整
- 动作是否匹配含义词
- 是否适合微信小尺寸
- 是否有明显脏边或错误文字
- 是否有明显违规或侵权风险

质检失败的图不应该直接作为成功结果交付给用户。

### 9. 导出基础投稿包

用户完成 8 张表情图后，可以免费导出一次基础投稿 ZIP。

基础投稿包包含：

- 8 张表情图
- 8 张缩略图
- 封面图
- 聊天面板图标
- 详情页横幅
- 含义词文件

不做自动投稿微信，不承诺自动上架。

## 积分规则

第一版采用积分制，不按项目赠送免费额度。

原则：

- 只要生成可见图片，就正常扣积分。
- 免费阶段只给方案，不给完整可用图片。
- 生成图片收费，基础交付免费。
- 完成 8 张后的首次基础投稿包导出免费。

| 动作 | 积分 |
|---|---:|
| 生成表情方案 | 0 |
| 生成 1 张角色候选图 | 3 |
| 生成 1 张表情图 | 3 |
| 单张重生成 | 3 |
| 完成 8 张后的首次基础导出 | 0 |
| 再次导出 | 2 |

完整一套基础消耗：

- 3 张角色候选：9 积分
- 8 张表情图：24 积分
- 首次导出：0 积分
- 合计：33 积分

## API 和外部能力

MVP 使用商业生图 API，不自部署模型。

需要接入：

- 文本模型 API：生成方案、角色卡、优化 prompt。
- 生图模型 API：生成角色候选和表情图。
- 图片存储：保存生成结果。
- 图片处理：生成缩略图、封面、图标、横幅。
- ZIP 打包：导出投稿包。
- 支付/积分系统：充值、扣积分、记录流水。

生图 API Key 放在服务端环境变量中，不能暴露到前端。

## 第一版页面

原型先覆盖 5 个页面：

1. 创建页：输入主题，查看积分余额，生成方案。
2. 方案确认页：编辑主角、画风、含义词和动作描述。
3. 角色选择页：生成并选择 3 张角色候选。
4. 表情生成页：分批生成 8 张，支持单张重生成。
5. 导出页：检查完整性，下载基础投稿 ZIP。

## MVP 不做

- 表情包名称生成
- 上传参考图
- 照片转表情
- 动态 GIF
- 自动投稿微信
- 微信账号登录/授权
- 表情包介绍生成
- 高级审核报告
- 批量生成多套
- 会员系统
- 素材市场
- 社区广场
- 多平台导出
- 复杂图片编辑器
- 自己部署生图模型

## 后续版本方向

第二版可以考虑：

- 上传角色参考图
- 保存角色资产
- 用同一角色生成第二套表情
- 动态 GIF
- 高级审核检查
- 投稿文案优化
- 多平台导出

## 成功标准

- 用户能用文字完成一套 8 张表情。
- 8 张看起来像同一个角色。
- 用户不需要频繁重生成。
- 用户能理解每次积分消耗。
- 完成 8 张后能拿到基础投稿包。
- 免费阶段不会给出可直接白嫖的完整图片。
